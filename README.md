# TGbots — Telegram-бот для адвокатской коллегии

Telegram-бот для автоматизации документооборота адвокатской коллегии: регистрация соглашений с доверителями и адвокатских запросов, ведение журнала в Excel, логирование действий пользователей.

---

## Возможности

- 📄 **Регистрация соглашений** — автоматическая нумерация (`С/мм/гг-`), даты, сведения о доверителе, характер поручения
- 📨 **Регистрация адвокатских запросов** — аналогичная нумерация (`З/мм/гг-`), наименование адресата
- 👥 **Роли пользователей** — первый зарегистрированный становится администратором
- 📊 **Excel-журнал** на Яндекс.Диске (`journal_registration.xlsx`) с двумя листами
- 💾 **База пользователей** в JSON на Яндекс.Диске
- 📝 **Логирование действий** в файл `actions.log` на Яндекс.Диске
- 🔄 **ConversationHandler** — пошаговые диалоги с возможностью отмены

## Стек

- **Python 3.10+**
- **[python-telegram-bot](https://python-telegram-bot.org/)** — взаимодействие с Telegram API
- **[yadisk](https://pypi.org/project/yadisk/)** — работа с Яндекс.Диском
- **[openpyxl](https://openpyxl.readthedocs.io/)** — генерация и редактирование `.xlsx`
- **[pandas](https://pandas.pydata.org/)** — обработка данных
- **[python-dotenv](https://pypi.org/project/python-dotenv/)** — загрузка переменных окружения

## Переменные окружения

Создайте файл `.env` в корне (или задайте переменные в окружении):

| Переменная             | Описание                                      |
| ---------------------- | --------------------------------------------- |
| `BOT_TOKEN`            | Токен Telegram-бота от [@BotFather](https://t.me/BotFather) |
| `YANDEX_OAUTH_TOKEN`   | OAuth-токен Яндекс.Диска                     |
| `YANDEX_CLIENT_ID`     | Client ID приложения Яндекс                   |
| `YANDEX_CLIENT_SECRET` | Client Secret приложения Яндекс               |

> ⚠️ Никогда не коммитьте `.env` в репозиторий. `.gitignore` уже настроен.

Приложение упадёт с понятной ошибкой при старте, если какая-либо переменная не задана.

## Локальный запуск

```bash
# 1. Клонировать репозиторий
git clone https://github.com/kgbspyfan-ship-it/TGbots.git
cd TGbots

# 2. Создать и активировать виртуальное окружение
python -m venv venv
source venv/bin/activate        # Linux/macOS
# venv\Scripts\activate         # Windows

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Создать .env с переменными (см. таблицу выше)

# 5. Запустить бота
python bot.py
```

## Деплой на Heroku

`Procfile` уже настроен. Достаточно:

1. Создать приложение в Heroku
2. Подключить этот репозиторий
3. Задать Config Vars в настройках приложения:
   - `BOT_TOKEN`
   - `YANDEX_OAUTH_TOKEN`
   - `YANDEX_CLIENT_ID`
   - `YANDEX_CLIENT_SECRET`
4. Включить автоматический деплой из ветки `main`

## Использование

1. Найдите бота в Telegram и нажмите **Start** (`/start`)
2. Введите ФИО для регистрации
3. Из главного меню выберите `📄 Соглашение` или `📨 Адвокатский запрос`
4. Следуйте инструкциям бота
5. Все записи автоматически сохраняются в Excel-журнал на Яндекс.Диске

Первый зарегистрированный пользователь получает роль **администратор**.
